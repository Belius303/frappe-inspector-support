## Critical Bug Reward

The reporter of a confirmed critical bug may receive one month of
Frappe Inspector Pro at no cost.

A qualifying report must:

- Affect the latest publicly available version
- Include clear and reproducible steps
- Demonstrate serious technical impact
- Be submitted privately first if it involves security or license bypass
- Not rely on unsupported IDE modifications or a deliberately corrupted environment
- Not already be known or previously reported

Examples of potentially qualifying impact:

- Exposure or transmission of private source code
- Destructive modification of project files
- Reproducible major IDE instability caused by the plugin
- A valid bypass of Pro feature enforcement
- Another issue of comparable severity

Ordinary exceptions, cosmetic defects, false-positive diagnostics, missing
features and minor performance issues are not considered critical.

The final severity assessment and reward eligibility are determined after
reproducing and confirming the report.

Do not post secrets, private source code, customer data, database dumps, API keys, or paid-license information in public issues.
